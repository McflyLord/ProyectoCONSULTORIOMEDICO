Requerimientos Funcionales: Clase Paciente: - El sistema debe permitir el registro de nuevos pacientes. - El sistema debe permitir la validacion de los datos de los pacientes. - El sistema debe permitir la editar datos de pacientes. - El sistema debe permitir la eliminación de pacientes. - El sistema debe permitir la búsqueda de pacientes por DNI. Clase Medico: - El sistema debe permitir el registro de medicos. - El sistema debe permitir la validacion de los datos de los medicos. - El sistema debe permitir la editar datos de medicos. - El sistema debe permitir la eliminación de medicos. - El sistema debe permitir la búsqueda de medicos por DNI. - El sistema debe permitir la asignación de horarios a médicos Clase HorarioMedico: - El sistema debe permitirla selección de horario para consultas Clase Cita: - El sistema debe permitir crear citas - El sistema debe permitir eliminar citas. Clase Historial: - El sistema debe permitir el registro del historial médico. - El sistema debe permitir el acceso al historial médico. Clase Registro: - El sistema debe permitir añadir observaciones. - El sistema debe permitir añadir recetas. - El sistema debe permitir añadir tratamientos. Clase Observación: - El sistema debe permitir registrar una observaciòn del médico. Clase Receta: - El sistema debe permitir registrar recetas del médico. Clase Tratamientos: - El sistema debe permitir registrar tratamientos del médico.
